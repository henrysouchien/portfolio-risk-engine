# Planning Docs Index

This folder tracks implementation plans, active work items, and historical planning records.

## Start Here

- [`BACKLOG.md`](./BACKLOG.md) - prioritized queue of planned work
- [`TODO.md`](./TODO.md) - immediate execution checklist
- [`PROGRESS.md`](./PROGRESS.md) - status updates and recent milestones
- [`BUGS.md`](./BUGS.md) - active planning-level bug tracker

## Folder Layout

- `./` (root) - active plans and current planning docs
- [`completed/`](./completed/) - finished plans and historical references
- [`performance-actual-2025/`](./performance-actual-2025/) - realized-performance investigation workstream
- [`investment-planning-reference/`](./investment-planning-reference/) - planning domain references
- [`baselines/`](./baselines/) - baseline artifacts used by plan documents

## Conventions

- Keep in-progress plans in this root folder.
- Move completed plans to [`completed/`](./completed/) once fully shipped.
- Prefer descriptive names ending in `_PLAN.md` for implementation plans.
- Use relative links so docs render correctly in local previews and GitHub.
